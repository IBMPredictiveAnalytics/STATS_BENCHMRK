Benchmark Statistics Jobs

This command runs one or two syntax files one or more times and records various statistics on computer resource consumption.

Syntax file to benchmark Select a syntax file to be benchmarked.

Alternate syntax file for comparison Optionally select another syntax file to be run in alternation with the first one. This is useful when you have two version of syntax and want to compare their resource use.

Processes to Measure Check boxes for the Statistics processes to be measured. These can be the Statistics frontend, the backend, and the Python and R programmability processes.

• The names of the processes vary some among Statistics versions. If you cannot specify a process you want to monitor, paste the syntax and edit the name.

Start the R process The R process must be started before running the benchmark command if it is to be measured. Check this box if that process has not been started, and you want the dialog box to do this.

• If the R Essentials are not installed, do not check this box.

Output File to Create Specify a file name to receive the recorded statistics. The file will be in csv format and can be read using File>Read Text Data. It will contain variable names on the first line. The variables include the process name, the cmdset, and the repetition number and the statistics values at the start and stop of each syntax file run. There is one record for each process.

Number of repetitions Specify the number of times that the syntax files should be run.

Statistics

Choose the particular statistics to be measured each time a syntax file is run.

All statistics Check this box to generate all available statistics.

• The other checkboxes are ignored if this box is checked.

• Checking this box generates some statistics not listed elsewhere in this dialog.

Time The total time for a job is the sum of the User and Kernel time. The process creation time is often not meaningful.

I/O In addition to the number of reads and writes, by using syntax the bytes counts are also available.

Memory The working set size tells you the number of pages in the process' memory. The page fault statistics tell you how many times the system had to retrieve code and other resources from disk because it was not in memory when needed.

Additional Features

This dialog generates syntax for the STATS BENCHMRK extension command. To display help for this command, run the following syntax:

STATS BENCHMRK /HELP.

Requirements

This command requires the Python Essentials, the Python Windows extensions, and the STATS BENCHMRK extension command. You can get the Python Windows extensions from Sourceforge. Be sure to get the version appropriate for the version of Python required by Statistics. You can download the other materials from the SPSS Community.
